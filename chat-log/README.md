# chat-log

โฟลเดอร์นี้ใช้เก็บบันทึกการทำงานของ AI Agent เพื่อให้ Agent คนถัดไปอ่านต่อและสานงานได้ถูกต้อง

## วัตถุประสงค์

ทุกครั้งที่ AI Agent ทำงานเสร็จ ให้สร้างไฟล์ chat-log ใหม่ไว้ในโฟลเดอร์นี้เสมอ เพื่อสรุปสิ่งที่ทำ, สิ่งที่พบ, สิ่งที่ยังค้าง, และขั้นตอนถัดไป

chat-log ต้องบอกบริบทให้ครบถ้วนทั้งโปรเจกต์ ไม่ใช่เฉพาะงานที่ทำในรอบนั้น เพื่อให้ Agent คนถัดไปอ่านแล้วต่อเนื่องได้ทันที และไม่หลุดงานเดิม

## หลักการตั้งชื่อไฟล์

ใช้รูปแบบ:

`{agent}_{YYYY.MM.DD}_{ลำดับ}.md`

ตัวอย่าง:

- `copilot_2026.06.13_001.md`
- `claude_2026.06.13_002.md`

## สิ่งที่ควรเขียนใน chat-log

ควรมีอย่างน้อย:

1. วัตถุประสงค์ของงานรอบนั้น
2. สรุปสิ่งที่ดำเนินการแล้ว
3. ไฟล์ที่เกี่ยวข้องหรือถูกแก้ไข
4. ปัญหาที่พบและวิธีแก้
5. สถานะปัจจุบัน
6. งานที่ยังค้างหรือคำถามที่ต้องส่งต่อ
7. ขั้นตอนถัดไปสำหรับ Agent คนต่อไป

8. ภาพรวมโปรเจกต์ที่เกี่ยวข้องกับงานรอบนั้น เช่น เอกสารหลัก, โครงสร้างข้อมูล, หรือขอบเขตที่ต้องคงไว้

9. รูปแบบไฟล์งานที่ใช้จริง เช่น ชื่อไฟล์, โฟลเดอร์, เวอร์ชัน, และวันที่ เพื่อให้ตามรอยได้ง่าย

## แนวทางการเขียน

- เขียนให้สั้นแต่ครบ
- ใช้ภาษาที่อ่านต่อได้ง่าย
- ระบุชื่อไฟล์และโฟลเดอร์ให้ชัด
- ถ้ามีข้อสรุปสำคัญ ให้บันทึกไว้ชัดเจนเพื่อไม่ให้ต้องไล่อ่านใหม่ทั้งหมด
- เขียนในรูปแบบไฟล์งานที่ต่อเนื่องได้จริง โดยเก็บลำดับเหตุการณ์, สิ่งที่ตัดสินใจ, และสิ่งที่ต้องส่งต่อไว้ครบ
- ถ้างานกระทบทั้งโปรเจกต์ ให้ระบุผลกระทบต่อส่วนอื่นด้วย ไม่เขียนเฉพาะเฉพาะจุดที่แก้

## เป้าหมายของการมี chat-log

- ให้ AI Agent คนถัดไปเข้าใจบริบทได้เร็ว
- ลดการทำงานซ้ำ
- รักษาเส้นทางการตัดสินใจและงานที่ค้างไว้
- ทำให้การส่งต่องานเป็นระบบและตรวจสอบย้อนหลังได้
- ป้องกันการหลุดบริบทของโปรเจกต์โดยรวม
- ทำให้ไฟล์งานแต่ละรอบเป็นหลักฐานการทำงานที่ต่อกันได้อย่างเป็นระบบ
